SpringJDBC基础

SpringJDBC简介

Spring的JDBC模块由四个包组成：core(核心包)、object（对象包）、dataSource（数据源包）、support（支持包）

Spring对JDBC的增强主要体现在它封装了传统的JDBC并提供了JDBCTemplate类，所以这个类是SpringJDBC的核心类

JDBCTemplate类

JDBCTemplate类包含了所有操作数据库的基本方法，包括增、删、改、查等

JDBCTemplate类提供的方法太多了，我在这里懒得打了，自行百度即可。

使用JDBCTemplate类完成简单程序

package io.gitee.yunchongxiao;

import com.mchange.v2.c3p0.ComboPooledDataSource;
import org.springframework.jdbc.core.JdbcTemplate;

public class TestJDBCTemplate01 {
    public static void main(String[] args) throws Exception {
        ComboPooledDataSource dataSource = new ComboPooledDataSource();
        dataSource.setDriverClass("com.mysql.cj.jdbc.Driver");
        dataSource.setJdbcUrl("jdbc:mysql://localhost:3306/chapter09?useSSL=true&useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai");
        dataSource.setUser("root");
        dataSource.setPassword("xyc@444049406");

        JdbcTemplate jdbcTemplate = new JdbcTemplate(dataSource);
        String sql = "insert into student (sname, age, course) values (?, ?, ?)";
        int result = jdbcTemplate.update(sql, "WangWu", 21, "Java");
        if (result > 0) {
            System.out.println("成功添加" + result + "条数据");
        } else {
            System.out.println("数据添加失败");
        }
    }
}

运行结果

成功插入1条数据

在Spring中管理JDBCTemplate类

上面的代码是我们自己手动创建了JDBCTemplate类，实际上，为了利用SpringIOC的优良性质，完全可以将其添加到Bean容器中自动管理。

将JDBCTemplate类配置到Spring容器中主要分两步，首先是在Spring容器中注册数据源

其次是在Spring容器中注册JDBCTemplate类，然后向JDBC类注入数据源，此时，Spring容器可以直接管理JDBCTemplate类。

在Spring中注册数据源

JDBCTemplate类依赖的数据源既可以是Spring内部提供的，也可以是第三方连接池，以c3p0为例，常用的属性如下表所示：

属性 描述
driverClass 数据库的驱动名称
jdbcUrl 连接数据库的路径
user 数据库用户名
password 数据库密码
maxPoolSize 最大连接数
minPoolSize 最小连接数
initialPoolSize 初始化连接数
maxIdleTime 连接的最大空闲时间

jdbc.driver = com.mysql.cj.jdbc.Driver

applicationContext.xml

<?xml version="1.0" encoding="UTF-8" ?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
            http://www.springframework.org/schema/beans/spring-beans.xsd
            http://www.springframework.org/schema/context
            http://www.springframework.org/schema/context/spring-context.xsd">

    <context:property-placeholder location="classpath:jdbc.properties"/>
    <bean name="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
        <property name="driverClass" value="${jdbc.driver}"/>
        <property name="jdbcUrl" value="${jdbc.url}"/>
        <property name="user" value="${jdbc.username}"/>
        <property name="password" value="${jdbc.password}"/>
    </bean>
    <bean name="jdbcTemplate" class="org.springframework.jdbc.core.JdbcTemplate">
        <property name="dataSource" ref="dataSource"/>
    </bean>
</beans>

JDBCTemplate类实现DQL操作

在JDBCTemplate提供了query()、queryForObject()、queryForList()方法来提供查询操作

查询单条记录时，常用query()方法或queryForObject()方法，

查询多条记录时，常使用query()或queryForList()方法。

当处理结果集时，如果要将返回的结果映射为开发人员自定义的类，则需要使用RowMapper<T>接口。

查询单条结果

查询单挑结果一般通过queryForObject()实现，如果查询结果时Java的基本类，则可以直接指定返回值的类型，

如果查询的结果要映射为用户的自定义类，则返回值的类型是RowMapper<用户自定义类>

查询结果是Java基本类类型

package io.gitee.yunchongxiao;

import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;
import org.springframework.jdbc.core.JdbcTemplate;

public class QueryOne {
    public static void main(String[] args) {

        ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
        JdbcTemplate jdbcTemplate = applicationContext.getBean("jdbcTemplate", JdbcTemplate.class);
        String sql = "select count(*) from student";
        Integer integer = jdbcTemplate.queryForObject(sql, Integer.class);
        System.out.println("查询结果为: " + integer);

    }
}
运行结果
查询结果为: 6

这里的查询结果是整形Integer，如果查询结果是字符串String应该也可以使用String.class获取。

查询结果要映射为自定义类

在实际开发中，更多的需求是将结果集之中的字段值映射为自定义类的属性，此时就需要RowMapper<T>接口

package io.gitee.yunchongxiao.bean;

import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;
import org.springframework.jdbc.core.BeanPropertyRowMapper;
import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.jdbc.core.RowMapper;

public class TestRowMapper {
    public static void main(String[] args) {
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
        JdbcTemplate jdbcTemplate = applicationContext.getBean("jdbcTemplate", JdbcTemplate.class);
        String sql = "select * from student where sid = 1";
        RowMapper<Student> rowMapper = new BeanPropertyRowMapper<Student>(Student.class);
        Student student = jdbcTemplate.queryForObject(sql, rowMapper);
        System.out.println(student);
    }
}
运行结果
Student{sid=1, sname='ZhangSan', age='20', course='Java'}

可以看到，查询结果是一个数据行，通过RowMapper映射为自定义类。

查询多条结果

查询多条结果则需要使用query()方法。

package io.gitee.yunchongxiao.bean;

import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;
import org.springframework.jdbc.core.BeanPropertyRowMapper;
import org.springframework.jdbc.core.JdbcTemplate;

import java.util.List;

public class TestQuery {
    public static void main(String[] args) {
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
        JdbcTemplate jdbcTemplate = applicationContext.getBean("jdbcTemplate", JdbcTemplate.class);
        BeanPropertyRowMapper<Student> rowMapper = new BeanPropertyRowMapper<Student>(Student.class);
        String sql = "select * from student where course = 'Java'";
        List<Student> query = jdbcTemplate.query(sql, rowMapper);
        for (Student stu : query) {
            System.out.println(stu);

        }
    }
}

Spring JDBC完成Dao封装

关于Dao的概念这本书并没有解释，通过百度，初步认定Dao为（Data Access Object），数据库对象接口的意思。

百度百科中认为一个典型的Dao实现由如下部分组成：

  1. 一个DAO工厂类；

  2. 一个DAO接口；

  3. 一个实现DAO接口的具体类；

  4. 数据传递对象（有些时候叫做值对象）.

我将它理解为一个将数据库的消息映射为Java对象的一个接口，其实我感觉和之前的ORM（Object Relational Mapping）的概念差不多

无论怎样，看看代码如何实现的吧

通过继承JDBCDaoSupport类的方式

除了将JDBCTemplate类对象直接注入Dao类之外，还可以使用继承JDBCDaoSupport类的方式实现Dao封装。

UserDaoSupport.Java

该类继承的JDBCDaoSupport类，同时实现了UserDao的接口。

package io.gitee.yunchongxiao.Dao;

import io.gitee.yunchongxiao.bean.Student;

    public int insert(Student student) {
        String sql = "insert into student (sname, age, course) values (?, ?, ?)";
        return getJdbcTemplate().update(sql, student.getSname(), student.getAge(), student.getCourse());
    }

    public void update(Integer sid) {

    }

    public void update(Student student) {

    }

    public void delete(Integer sid) {

    }

    public List<Student> selectAll() {
        BeanPropertyRowMapper<Student> rowMapper = new BeanPropertyRowMapper<Student>(Student.class);
