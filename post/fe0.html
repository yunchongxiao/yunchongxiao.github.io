【操作系统】1 计算机系统概述

发表于2022-07-10|更新于2022-07-11|2023年操作系统考研复习指导
|字数总计:2.7k|阅读时长:8分钟|阅读量:|评论数:

操作系统是计算机系统中最基本的系统软件

操作系统四个特征

  1. 并发

    并发：同一段时间间隔（通过分时实现）
    并行：同一时刻（通过多处理机实现）

  2. 共享

    资源供多个并发程序共同使用

    1. 互斥共享：独占资源，如打印机
    2. 同时访问：允许并发访问，分时共享，如磁盘设备

  3. 虚拟

    虚拟：将一个物理实体变为多个逻辑上的虚拟体

    • 虚拟处理器（时分复用）
    • 虚拟存储器（空分复用）

  4. 异步

    异步：程序并发执行中，由于资源有限，程序断断续续执行，以不可预知的速度向前推进
    需要保证重复运行进行能得到相同的结果

操作系统的功能

资源管理者

  1. 处理机管理

    处理机的分配和运行以进程、线程为基本单位，因此处理机管理可归结为进程管理

    1. 进程创建
    2. 进程撤销
    3. 进程调度
    4. 资源共享

  2. 存储器管理

    1. 内存分配与回收
    2. 地址映射
    3. 内存保护与共享
    4. 内存扩充

  3. 文件管理

    计算机的信息都已文件形式存在

    1. 文件存储空间管理
    2. 目录管理
    3. 文件读写与保护

  4. 设备管理

    主要任务是完成I/O请求

    1. 缓冲管理
    2. 设备分配
    3. 设备处理
    4. 虚拟设备

用户与计算机硬件系统的接口

  1. 命令接口

    命令接口分为联机命令接口和脱机命令接口

    1. 联机命令接口：交互式命令接口（分时、实时系统）
    2. 脱机命令接口：批处理命令接口

  2. 程序接口==（系统调用组成）==

    程序接口由系统调用（广义指令）组成，用户通过系统调用来获取服务

操作系统发展历程

  1. 手工操作阶段

    无操作系统，存在两个重大缺点

    1. 用户独占主机
    2. CPU等待手工操作，利用不充分

  2. 批处理阶段

    存在操作系统，无法进行人机交互

    1. 单道批处理系统：程序运行中CPU需要等待低速I/O完成，利用率不高
    2. 多道批处理系统：资源利用率高，但没有人机交互

  3. 分时操作系统

    将处理器的实践分成很短的时间片，将时间片轮流交给各作业使用

    缺点：不能保证规定时间完成

  4. 实时操作系统

    1. 硬实时：必须在规定时间完成，如导弹拦截
    2. 软实时：允许偶尔超时，如抢票系统

  5. 网络操作系统

    将多台电脑结合，实现各台电脑的数据传送，强调资源共享

  6. 分布式计算机系统

    若干计算机协同完成同一任务，无主从计算机之分

  7. 个人操作系统

    目前最广泛：Windows, Linux

操作系统运行环境（处理器模式）

特权指令与核心态

核心态：管太、内核态，可以执行特权指令
特权指令：I/O指令、置中断指令、存取用于内存保护的寄存器、送程序状态字到程序状态字寄存器等指令

非特权指令和用户态

用户态：目态、只能执行非特权指令

内核功能

  1. 时钟管理

    计时，显示正确的系统时间
    计量，例如管理时间片，度量作业运行程度

  2. 中断机制

    仅有一小部分功能属于内核：

    • 保护中断现场信息
    • 恢复中断现场信息
    • 转移控制权

  3. 原语

    1. 可公用的通用程序
    2. 最接近硬件，处于操作系统最底层
    3. 具有原子性，可以通过运行前关中断实现
    4. 运行时间短，调用频繁

  4. 管理系统控制的数据结构

    • 进程管理
    • 存储器管理
    • 设备管理

中断与异常的概念

  • 中断：来自CPU执行指令之外的命令

    I/O结束中断
    时钟中断

  • 异常：来自于CPU当前指令

    程序非法操作码
    地址越界
    运算溢出
    虚存系统缺页
    陷入指令（用来进入和心态，发起系统调用）

系统调用

操作系统结构

分层法

自下而上划分，最底层0层为硬件，最高层n层为用户接口

优点

便于调试和验证
容易扩充和维护

缺点

分层后不够灵活，难以确定每层的合理分界
效率差，需要自上到下穿越多层

模块化

将操作系统划分为独立的模块

优点

操作系统内核

宏内核

将系统的主要功能模块作为一个紧密联系的整体运行在和心态

优点：性能优越
缺点：设计难度大

Windows, Linux, macOS等主流操作系统都是宏内核

微内核

将内核中最基本的功能保留在内核中，其余功能移动到用户态执行

特点

  1. 足够小的内核
  2. 基于客户/服务器模式
  3. 采用机制与策略分离
  4. 采用面向对象技术

优点：降低内核的设计复杂性，易扩展，可移植，安全性和可靠性==（用户态服务崩溃不会导致核心态也崩溃）==，可分布式计算

缺点：需要频繁在用户态和核心态频繁切换，性能不高

设计模式

微内核架构 = 微内核 + 多个服务器
微内核：精心设计的，能够实现操作系统最基本功能的小型内核，包括：

与硬件处理紧密相关的部分
一些基础功能，如：

进程线程管理，如调度、切换、同步等
低级存储器管理，如依赖硬件的地址变换，
中断和陷入处理
客户和服务器之间的通信

多个服务器：操作系统的绝大数其余功能都放在一组运行在用户态的服务器（进程）中，如对用户进程进行分类、虚拟存储器置换算法

外核

不理解

操作系统引导

  1. 激活CPU

    激活的CPU读取ROM中的boot程序
    将BIOS的第一条指令放入指令寄存器开始执行

  2. 硬件自检

    硬件自检故障，则发出嗡鸣声，终止启动

  3. 加载带有操作系统的硬盘

    将带有操作系统的硬盘数据加载到内存中

  4. 加载主引导记录MBR

    引导CPU找到操作系统所在的主分区

  5. 扫描硬件分区表

    扫描分区，寻找操作系统

  6. 加载分区引导记录PBR

    从分区中找到操作系统

  7. 加载启动管理器

    找到操作系统的启动管理器

  8. 加载操作系统

虚拟机

第一类虚拟机：可以提供多个虚拟机

第二类虚拟机：单纯的进程，如VMWare

错题整理

缓存管理由操作系统自动实现，不对用户开放

操作系统与用户通信接口通常不包括（D）

A:shell B：命令解释器 C:广义指令 D:缓存管理指令

缓存管理由操作系统自行完成，因此不提供给用户
A：shell，命令解析器，属于命令接口
B：命令解释器：属于命令接口
C：广义指令：属于系统调用

处理机、通道的知识点

单处理机系统中,可并行的是（BCD）

A：进程与进程 B：处理机与设备 C：处理机与通道 D：设备与设备

操作系统提供系统调用，库函数使用系统调用，属于用户程序

开机后，操作系统被加载到RAM

操作系统 单用户多用户如何划分

有点绕

【2016统考真题】下列关于批处理系统的叙述中，正确的是（BC）

A. 批处理系统允许多个用户与计算机直接交互
B. 批处理系统分为单道批处理系统和多道批处理系统
C. 中断技术使得多道批处理系统的I/O设备可与CPU并行工作

虽然中断技术不能体现并行，但是当CPU处理I/O请求时，若不存在中断技术，则CPU需要等待I/O完成，而中断后，CPU可以在I/O执行时转去运行另一道程序，这就体现了并行工作。

下列操作系统功能，不需要硬件支持的是（）

A. 进程调度 B. 时钟管理 C.地址映射 D.中断系统

时钟管理需要硬件计数器保持时钟的运行
进程调度由调度算法决定，不需要硬件支持
地址映射愮基地址即偏移地址等硬件支持

用户态到和心态由硬件完成

广义指令又叫系统调用

执行系统调用的主要步骤

  1. 传递系统调用参数
  2. 执行陷入指令
  3. 执行相应的服务程序
  4. 返回用户态

不同操作系统提供的程序调用接口不同
