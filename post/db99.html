第一章-MyBatis概述

|字数总计:3.1k|阅读时长:15分钟|阅读量:|评论数:

传统JDBC的劣势

  • JDBC是Java程序实现数据访问的基础，它提供了一套操作数据库的API，
    • 加载驱动
    • 获取链接
    • 获取执行对象
    • 发送SQL语句
  • 但存在以下的局限性：
    • 代码繁琐
    • 表关系维护复杂
    • 硬编码
    • 性能问题
  • 所以企业通常使用ORM来完成数据库的操作

ORM简介

ORM是Object Relational Mapping，即关系对象映射，是一种规范。

ORM将简单Java对象（POJO，Plain Ordinary Java Object）和数据库表记录进行映射。

这样开发人员就可以通过操作Java对象来对数据库进行操作

ORM的主流框架有Hibernate、MyBatis

MyBatis简介

MyBatis是一款优秀的ORM框架，它支持自定义SQL，存储过程以及高级映射。

它几乎避免了所有的JDBC代码和手动设置参数以及获取结果集

它利用XML文件或者注解进行配置和映射

将接口和POJO映射成数据库中的记录

MyBatis功能架构

  • API接口层
  • 数据处理层
  • 基础支撑层

API接口层

API接口层对开发人员提供操作数据库的接口，在收到开发人员的请求时，会调用数据处理层进行具体的数据处理。

数据处理层

负责具体的参数映射，SQL解析，SQL执行和结果映射，它的功能就是根据请求完成一次数据库操作。

基础支撑层

负责最基础的功能，包括连接管理，数据管理，配置加载缓存处理等。

MyBatis的工作流程

  1. 读取配置文件和映射文件
  2. 生成SqlSessionFactory对象
  3. 通过SqlSession操作数据库

读取配置文件和映射文件

其中配置文件设置了数据源，事务等信息，

映射文件设置了SQL执行相关的信息。

生成SqlSessionFactory对象

SqlSessionFactory对象的重要功能就是创建MyBatis的核心对象SqlSession

通过SqlSession操作数据库

SqlSession中封装了操作数据库的所有方法，开发人员通过调用SqlSession中的方法来操作数据库

MyBatis的重要API

  1. SqlSessionFactory
  2. SqlSession

SqlSessionFactory

  • SqlSessionFactory的首要功能就是创建SqlSession对象
  • 因此，每一个MyBatis应用都以SqlSessionFactory
  • 所以SqlSessionFactory会存在于MyBatis应用的整个生命周期
  • 因此一个MyBatis应用一般只使用一个SqlSessionFactory对象

创建SqlSessionFactory

  • SqlSessionFactory由SqlSessionFactoryBuilder对象创建

  • 通过使用SqlSessionFactoryBuilder对象的build()函数，由两种参数方式创建

    •   SQLSessionFactory build(InputStream inputStream[, String evironment][, properties props])

  这种方式需要预定义config，因此仅作了解即可

调用SqlSessionFactory

  • 创建SqlSessionFactory后，调用该对象的openSession()方法创建SqlSession对象
  • 该函数可以设置许多参数

SqlSession

  • SqlSession是MyBatis中的核心类对象，也会存在于MyBatis中的全生命周期
  • 因此在执行完操作后要及时关闭SqlSession对象，以免影响系统性能
  • SqlSession提供许多SQL执行函数，包括增删改查、事物的提交和回滚
