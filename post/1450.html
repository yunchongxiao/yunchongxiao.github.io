【操作系统】2-进程与线程

|字数总计:2.4k|阅读时长:7分钟|阅读量:|评论数:

进程

进程是程序的运行过程，是系统进行资源分配和调度的独立单位

进程特征

  1. 动态性，进程存在创建、活动、暂停、终止等过程，具有生命周期
  2. 并发性，多个进程同存与内存中，可以并发执行
  3. 独立性，进程实体可以独立执行，独立获得资源和调度
  4. 异步性，进程之间是异步的，因此需要同步机制保证正确

进程的状态

  1. 运行态：

    正在处理机上运行的状态

    从就绪态转向运行态，则分配处理机资源即可
    阻塞态不能直接转向运行态，需要先唤醒为就绪态

  2. 就绪态：

    获得处理机外的一切资源，获取处理机资源即可运行

  3. 阻塞态：

    缺少资源，即使获得处理机也不可运行
    若从运行态转到阻塞态（因资源不足主动让出处理机）：

    调用阻塞原语Block
    找到进程对应PCB
    保存现场信息，并将状态设置为阻塞态
    插入相应的阻塞队列
    转移处理机

  4. 创建态：

    申请空白PCB（有限，可能失败）
    向空白PCB写入控制和管理进程的信息
    分配进程所需资源
    根据资源是否满足转入就绪队列或阻塞队列

  5. 结束态

    将进程状态置为结束态，并结束子孙进程
    资源释放与回收
    删除PCB

进程的组织

程序段

调度到CPU中执行的代码段和赋值数据段，以及常量，可被多个进程共享

数据段

包括数据堆段、数据栈段。

堆：动态分配
栈：临时变量

进程控制块PCB

（Process Control Block，PCB）是进程存在的唯一标志，用来描述进程的基本情况和运行状态，常驻内存，随时存取。

常见内容

进程描述信息 进程控制管理信息 资源分配清单 处理机相关信息
进程标识符PID 进程当前状态 代码段指针 通用寄存器值
用户标识符UID 进程优先级 数据段指针 地址寄存器值
代码运行入口地址 堆栈段指针 控制寄存器值
程序外村地址 文件描述符 标志寄存器值
进入内存时间 键盘 状态字
处理机占用时间 鼠标
信号量使用

进程通信

共享存储

进程间存在可共同访问的共享空间，需要使用同步互斥工具保证正确性

消息传递

通过发送消息和接收消息两个原语实现。可直接通信或间接通信

管道通信

管道文件pipe，半双工通信，每次只允许读或写

线程

引入目的：减少并发时空开销，提高性能

组成

线程ID，程序计数器，寄存器组，堆栈等信息，保存在线程控制块中TCB中

线程与进程的比较

  • 调度：进程切换需要切换上下文，代价大，线程切换代价小，不需要切换进程
  • 并发：都可以并发，进一步提高资源利用率与系统吞吐量
  • 资源：线程无资源，进程有资源
  • 独立性：进程是系统资源的分配单元，线程是处理机的分配单元
  • 系统开销：线程开销小，进程开销大
  • 多处理机：线程可以分配到多个处理机，进程不行。

实现方式

用户级线程与内核级线程

用户级线程ULT

User-Level Thread，由用户管理线程，内核感受不到线程的存在，因此调度仍以进程为单位，
缺点：
若某进程的线程多，则其被分配的处理机时间就会变短
一个线程阻塞，其余线程也被阻塞

优点：
线程切换不需要通知内核，节省开销
可以自定义调度算法
实现与操作系统无关

常用线程库实现

内核级线程KLT

Kernel-Level Thread，内核管理线程调度

优点：
允许多处理机并行执行线程
线程阻塞时不影响其余线程

缺点：
切换需要进入核心态，开销大

多线程模型

一对一：一个用户级线程对应一个内核级线程，并发高，开销大

多对一：多个用户级线程对应一个内核级线程，切换快，开销小，一个阻塞全部阻塞

多对多：综合上述优缺点

处理机调度

调度的三级层次

高级调度（作业调度）：对于作业进行调入调出，每个作业仅调入和调出一次，表示作业的开始和结束。

中级调度（内存调度）：将进程调至外存，称之为挂起，调回内存称之为激活

低级调度（进程调度）：内存中的进程调度问题，频率很高。

调度目标

CPU利用率

CPU利用率=CPU有效工作时间CPU有效工作时间+CPU空闲等待时间CPU利用率 =\frac{CPU有效工作时间 }{CPU有效工作时间 + CPU 空闲等待时间}CPU利用率=CPU有效工作时间+CPU空闲等待时间CPU有效工作时间​

系统吞吐量

单位时间内CPU完成作业的数量，显然短作业优先可以提高吞吐量

周转时间

周转时间 = 作业完成时间 - 作业提交时间 = 作业等待时间 + 作业执行时间

平均周转时间

每个作业的周转时间总和除以作业数量

带权周转时间

带权周转时间=周转时间作业实际运行时间带权周转时间 = \frac{周转时间}{作业实际运行时间}带权周转时间=作业实际运行时间周转时间​

平均带权周转时间

每个作业的带权周转时间求和除以作业数量

等待时间

进程等待处理机的时间

响应时间

从提交到首次响应的时间，长用于评估交互式系统

调度实现

排队器

就绪进程通过排队器插入就绪队列

分派器

CPU通过分派器交给就绪进程

上下文切换器

进程切换时，保存当前进程上下文到PCB中，通过分派器选择新进程，并将其CPU现场信息装入处理机各个寄存器中

不进行调度的时机

处理中断的过程

中断过程较为复杂，不应被剥夺处理机资源

操作系统内核临界区

进入临界区后不应被打断，应该通过加锁的方式保护临界区

原子操作

原子性，不可打断。

调度方式

抢占式调度：分时、实时系统

非抢占式调度：批处理系统，开销小

闲逛进程

CPU没有任务时执行的进程，优先级最低

经典调度算法

先来先服务FCFS

先来先服务，顺序执行，不剥夺，长作业友好，短作业不友好，CPU繁忙友好，IO繁忙不友好

短作业优先SJF

平均等待时间最短、平均周转时间最短。

短作业友好，长作业不友好，可能饥饿，不考虑紧迫程度

优先级调度算法

抢占与非抢占

静态优先级与动态优先级

优先级设置准则：

  • 系统进程 > 用户进程
  • 交互进程 > 非交互进程
  • I/O进程 > 计算型进程

高响应比优先

响应比响应比RP=等待时间+要求服务时间要求服务时间响应比R_P = \frac{等待时间 + 要求服务时间}{要求服务时间}响应比RP​=要求服务时间等待时间+要求服务时间​

短作业响应比高，长作业随着等待时间的增多，响应比也会提高

时间片轮转调度

按先来先服务排队，每个进程用相等的时间片

多级队列调度

将进行分到不同队列中处理

多级反馈队列调度

时间片轮转 + 多级队列 + FCFS

  1. 从上至下多个就绪队列，优先级逐渐降低
  2. 每个队列的时间片不同，优先级越高，时间片越短
  3. 每个队列都采用FCFS，即先来先服务
  4. 时间片用完时进入下级队列的末尾，被抢占则进入本级队列末尾，最下级的任务进入本级的末尾
  5. 永远按照队列优先级调度，只有上层队列为空时，本层队列才会被调度

上下文切换

调度：决策行为，提出理论

切换：分配行为，进行实践

错题

下列叙述正确的是：A

A. 线程包含CPU现场，可以独立执行程序
B. 每个线程有自己独立的地址空间

线程被不同进程调用，是同一个线程

键盘驱动程序只需要一个线程就足够

HTTP请求可以多线程

用户登录需要为用户分配管理线程，设备分配不需要线程

父进程、子进程不能共享虚拟空间，也不能同时使用同一临界资源

IO操作

执行IO操作时，为阻塞态，IO结束后，由阻塞变为就绪
