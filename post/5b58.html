灵感总结

使用结点的度作为数值计算时，计算的该结点的孩子结点个数，因此需要考虑根结点，因为根节点不可能是孩子结点。

使用度的结点作为分类时，不需要考虑根节点，因为根节点也会被分到对应度的结点中。

超简单但不会做

一棵完全二叉树上有1000个结点，求叶子结点的个数

我们可以通过完全二叉树的性质简单的计算出结果：

  1. N = n0+n1+n2（N是结点总数）
  2. n0=n2+1
  3. n1=1或0（度为1的点仅有0个或1个）

联立上述三式：

1000 = 2n2 +n1 +1

进而得到n0=500

另一种更简单的方法

由于完全二叉树有1000个结点，因此最后一个结点的编号为1000，其父节点编号为1000/2=500，因此编号为500之后的结点都是叶子结点，一共有1000-500=500个结点

术语

  • 结点：一个数据元素
  • 结点的度：该结点的分支（子树）个数
  • 叶子结点：终端结点，度为0的结点
  • 层次路径：从根节点开始，到指定结点的路径（唯一）
  • 深度：结点的最大层次值，根节点的层为1
  • 有序树：子树具有次序
  • 森林：多棵树

计算性质

  1. 结点个数 = 所有结点的度之和加一

    树中任意结点（除根节点）都有一个分支指向他

  2. 对于度为m的树，定义叶子结点的个数为n0，度为1的结点个数为n1，……度为m的结点个数为nm，则有：
    n0=n2+2n3+3n4+...+(m−1)nm+1n_0 = n_2 + 2n_3 + 3n_4 + ... + (m-1)n_m + 1n0​=n2​+2n3​+3n4​+...+(m−1)nm​+1

    从结点本身考虑：树的总结点数为：n0+n1+n2+...+nmn_0+n_1+n_2+...+n_mn0​+n1​+n2​+...+nm​
    从节点的度（子树）考虑，树的总结点数为：
    0n0+1n1+2n2+3n3+...+mnm+10n_0+1n_1+2n_2+3n_3+...+mn_m+10n0​+1n1​+2n2​+3n3​+...+mnm​+1
    联立可得：
    n0=n2+2n3+3n4+...+(m−1)nm+1n_0 = n_2 + 2n_3 + 3n_4 + ... + (m-1)n_m + 1n0​=n2​+2n3​+3n4​+...+(m−1)nm​+1

  3. 度为m的树中，第i层最多有mi-1个结点

    第一层最多1个，第二层最多m个，第三层最多m2，…，第i层最多mi-1

  4. 度为m，高为h的树中，最多有mh−1m−1\frac{m^h-1}{m-1}m−1mh−1​个节点

    使用性质3，将每一层的最大结点树加和，即一个等比数列求和

  5. 具有n个结点的m度树的最小高度是：log⁡mn(m−1)+1\log_m^{n(m-1)+1}logmn(m−1)+1​

    使用性质4，逆推即可得到h

完全二叉树的性质

  1. n0=n2+1
  2. 结点总数=所有结点度之和+1
  3. 第i层，最多2i-1个结点
  4. 前i层，最多有2i-1个结点
  5. 深度为k的结点总数为n，满足2k≤n≤2k-1
  6. 高度为k的完全二叉树，叶子结点仅出现在最后两层（k，k-1层）
  7. n个结点的完全二叉树深度为log2n + 1
  8. 对于任意结点，右子树的最大层次为l，则左子树的最大层次为l或l+1
  9. 任意编号为i的结点，左孩子为2i，右孩子为2i+1
  10. 度为1的结点个数，只能是1或是0

哈夫曼树

哈夫曼树是最优二叉树，其相关概念如下：

  1. 结点路径：树的一个节点到另一个结点之间的路径
  2. 路径长度：结点路径的边数
  3. 权值：叶子结点的权值
  4. 结点的带权路径长度：叶子结点的权值×路径长度
  5. 数的带权路径长度：所有叶子结点的带权路径长度的总和

哈夫曼树只有度为0和度为2的结点
