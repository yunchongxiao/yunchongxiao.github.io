数据库简介

在Django中，使用SQLite作为数据库管理系统，这是一个简易型数据库管理系统，方便简洁，但只适合测试开发时使用，对于真正的大型数据库，七扩充性和性能就有了很大的限制，我们这里先使用SQLite进行学习，后续会教授如何使用更适合正式网站的数据库系统。

对于网站来说，最重要的内容就是数据库，所以设计网站的第一步往往是根据网站的需求先设计数据库，确定数据库的所有细节，并且做好正规化操作（如拆解范式）后，才能开始编写程序。

这个表有几个注意点：

  • 为了方便观察数据，我们最好定义一下__str__函数，这样我们在admin界面查看数据项时我们就能查看到实际的信息，如果不设置，数据项默认会显示（类名+编号）

  • Django对于每条数据，都会隐式的创建一个id字段，该字段作为主键来区别数据项，所以一般不需要自己设置主键，

  • Student与Teacher之间存在外键关系，因此我们可以看到student类中的teacher是通过ForeignKey进行定义的，后面的on_delete参数表示在删除外键时，对引用外键的数据的处理：

    1. on_delete=models.CASCADE：引用外键的数据一并删除
    2. on_delete=models.PROTECT：禁止删除并抛出一个错误
    3. on_delete=models.SET_NULL：将外键设置为null，前提是外键可以为Null
    4. on_delete=models.SET_DEFAULT：将外键设置为默认值，前提是外键设置了默认值
    5. on_delete=models.DO_NOTHING：do nothing.

  • 每次修改完models.py后，我们需要让数据库的数据表进行同步的修改，因此需要：

    1. python manage.py makemigrations：记录数据表的修改命令
    2. python manage.py migrate：执行数据表的修改命令
