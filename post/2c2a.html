快速排序

发表于2020-08-08|更新于2022-01-05|排序算法
|字数总计:693|阅读时长:2分钟|阅读量:|评论数:

代码简介

快速排序是一个很快的排序，据说是冒泡排序的改良版算法，其排序的常数项较小，速度很快，可以说是最快的排序算法之一，美中不足便是快排不具备稳定性。

代码思想

从数列里随机选取一个值作为基准值key，并将该值放在给定范围的数组的首位；
根据基准值将给定范围的数组分成两部分，左边的值均比基准值小，右边的值均比基准值大；
对已经分好的数组两边递归调用quick_sort函数即可。

代码实现

#include <iostream>
#include "my_sort.hpp"
#include <ctime>
#include <cstdlib>

int random(int max)
{
    int seed = time(0);
    srand(seed);
    return rand() % max;
}

// 分区函数，以左端点为基准值，将left~right之间的数分成两部分
int partation(int *array, int left, int right)
{
    int start = random(right - left);
    swap(array[left], array[start + left]);

    if (left >= right)
        return left;
    int key = array[left];
    for (int i = left + 1; i < right; i++)
    {
        if (array[i] < key)
        {
            swap(array[i], array[left + 1]);
            swap(array[left], array[left + 1]);
            left++;
        }
    }
    return left;
}

// 快速排序
void quick_sort(int *array, int left, int right)
{
    if (left < right) // 递归出口
    {
        int partation_index = partation(array, left, right);
        quick_sort(array, left, partation_index - 1);
        quick_sort(array, partation_index + 1, right);
    }
}

int main()
{
    int num_array[SIZE];
    init_array(num_array, SIZE);
    display(num_array, SIZE);
    quick_sort(num_array, 0, SIZE);
    display(num_array, SIZE);
}

算法评估

时间复杂度

快速排序的算法复杂度是O(nlogn)
在选择基准值时我们选择了随机基准值，因此在大数据样本中可以认为在分区的过程中能够使得数组均分成等长度的两端，
因此在递归的子问题中数据规模变为原来的一半，因此和最前所讲述的归并排序是一致的,这里借用归并排序的图像，实际上每层都是遍历一遍，进行有限次可能的交换。因此时间复杂度为O(nlogn)。

空间复杂度

快速排序的空间复杂度为O(1)
因为仅仅使用了有限个变量，排序过程是在原数组空间进行的，因此快速排序的空间复杂度为O(1)。

稳定性

快速排序是不稳定的。
由于这里需要将随机选定的基准值放到第一位，有这种大幅度的跨越，因此是不稳定的，若去掉随机化基准值操作，则可以达到稳定性。由于实际应用中快速排序都是随机化基准值的（可以理解问随机化基准值可以保证O(nlogn)的时间复杂度），因此快速排序是不稳定的。
