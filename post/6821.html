为了降低高并发访问给数据库带来的压力，大型企业项目中都会使用缓存

使用缓存可以减少磁盘IO、减少程序与数据库的交互，帮助程序迅速获取所需数据，提升系统响应速度

接下来对MyBatis的缓存处理机制做出详细讲解。

MyBatis的缓存机制（选学）

通常情况下，缓存将程序使用频率较高的数据存储在内存中，它具有可被快速读取和使用的特点。

MyBatis在内存中开辟一块区域，该区域用于保存程序对数据库的操作，和数据库返回的信息，

如果下一次在进行相同的操作，则直接返回该信息即可，而不必访问数据库重新获取数据

MyBatis支持的缓存分为一级缓存和二级缓存，

其中一级缓存是SqlSession级别的缓存。

  • 当创建SqlSession对象操作数据库时，MyBatis在SqlSession上引入一个HashMap对象作为存储数据的区域，

  • 该HashMap对象的key值是由Sql语句、条件、statement等信息组成的唯一值，

  • 该HashMap对象的value值是查询出来的结果对象

  • 不同的SqlSession对象拥有自己独立的HashMap

MyBatis的二级缓存是Mapper级别的缓存，

  • 多个SqlSession去操作同一个Mapper的Sql语句，二级缓存是跨越SqlSession的，多个SqlSession可以共用一个二级缓存
  • 和一级缓存类似，二级缓存HashMap对象的key值是由Sql语句、条件、statement等信息组成的唯一值，
  • HashMap对象的value值是查询出来的结果对象。

在实际应用中，一级缓存是默认开启的，二级缓存是需要手动开启。

一级缓存

一级缓存的原理

在MyBatis中，如果一级缓存开启，当同一个SqlSession对象多次执行完全相同的Sql语句时，在第一次执行完成后，MyBatis会将查询结果写入到一级缓存，

此后，如果程序没有执行插入、更新、删除操作，当第二次执行相同的查询语句时，MyBatis会直接读取一级缓存中的数据。

若程序执行了插入、更新、删除的操作，则MyBatis会清空一级缓存的内容，防止程序误读。

POJO类

Student.java

package io.gitee.yunchongxiao.pojo;

public class Student {
    private int sid;
    private String sname;
    private String age;
    private String course;

    public int getSid() {
        return sid;
    }

    public void setSid(int sid) {
        this.sid = sid;
    }

    public String getSname() {
        return sname;
    }

    public void setSname(String sname) {
        this.sname = sname;
    }

    public String getAge() {
        return age;
    }

    public void setAge(String age) {
        this.age = age;
    }

    public String getCourse() {
        return course;
    }

    public void setCourse(String course) {
        this.course = course;
    }

    @Override
    public String toString() {
        return "Student{" +
                "sid=" + sid +
                ", sname='" + sname + '\'' +
                ", age='" + age + '\'' +
                ", course='" + course + '\'' +
                '}';
    }
}

编写配置文件

StudentMapper.xml

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">


<mapper namespace="student">
    <select id="selectStudentBySid" parameterType="int" resultType="student">
        select * from student where sid = #{sid}
    </select>

    <update id="updateStudentBySet" parameterType="student">
        update student set sname = #{sname}, age = #{age} course = #{course}
    </update>

</mapper>

编写测试类

MyTest.java

package io.gitee.yunchongxiao.pojo;

import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import org.junit.Test;

import java.io.IOException;
import java.io.InputStream;

public class MyTest {


    public static SqlSession getSession() {
        String resource = "mybatis-config.xml";
        InputStream inputStream = null;
        try {
            inputStream = Resources.getResourceAsStream(resource);
        } catch (IOException e) {
            e.printStackTrace();
        }
        SqlSessionFactory sessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        return sessionFactory.openSession();
    }

    @Test
    public void selectStudentBySid() {
        SqlSession sqlSession = getSession();
        Student student1 = sqlSession.selectOne("selectStudentBySid", 4);
        System.out.println(student1);
        Student student2 = sqlSession.selectOne("selectStudentBySid", 4);
        System.out.println(student2);
        sqlSession.close();
    }
}

二级缓存

二级缓存的原理

  • 二级缓存的作用域是Mapper，与一级缓存相比，二级缓存的作用域范围更大，它可以跨越多个SqlSession对象，而且二级缓存可以自定义缓存源
  • 当开启二级缓存时，MyBatis以namespace区分Mapper，如果多个SqlSession对象使用同一个Mapper执行相同的查询语句时，MyBatis会直接读取二级缓存中的数据。
  • 当执行了增删改的操作后，二级缓存会被清空，防止程序误读。
  • 二级缓存需要手动开启

二级缓存的应配置

开启二级缓存的全局配置

<setting>
	<setting name="cacheEnabled" value="true"/>
</setting>
属性名称 描述
eviction 收回策略，包括LRU、FIFO、SOFT、WEAK四个可选值
flushInterval 刷新间隔，默认为不刷新
size 引用数据
readOnly 缓存设置为只读时，性能高，设置为可读写时，安全性高

EhCache缓存

EhCache简介

在实际开发中，许多项目都采用分布式的系统架构，分布式系统架构就是将项目拆分成若干子项目并使他们协同发挥功能的方式。

在分布式系统架构下，为了提升系统性能，通常会采用分布式缓存对缓存数据进行集中管理。EhCache就是实现该目标的

EhCache是一种获得广泛应用的开源分布式缓存框架，具有快速、简单、能够提供多种缓存策略等特点
