React main concepts（四）

发表于2021-10-04|更新于2022-01-31|react guide to main concepts.
|字数总计:1.5k|阅读时长:6分钟|阅读量:|评论数:

Components and Props

组件旨在将 UI 分离成许多独立、可复用的小片段，并可以独立的设计它们。从概念上看，组件像是 JavaScript 中的函数，它接受任意的输入（props），并返回一个 React 元素，用来描述页面中欲呈现的内容。

Function and Class Components

最简单的定义组件的方式是函数组件（Function Components），该类组件接受一个对象参数props，并返回一个 React 元素：

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

我们也可以使用 ES6 class 来创建一个类组件（Class components）：

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>
  }
}

从 React 的角度看，这两类组件是等价的。关于两类组件各自的特性我们会在下一章节进行讲解。

Rendering a Component

在之前，我们遇到的 React 元素代表的都是 DOM 标签，如：

const element1 = <h1>Hello, world</h1>;
const element2 = <div>...</div>

然而，React 元素同样可以代表用户定义的组件：

const element = <Welcome name="Sara"/>

当 React 元素代表用户定义的组件时，它将 JSX 的属性及其子元素当作一个单对象（props）传递给该组件，我们可以访问该对象进行带取值：

  • 函数组件

    function Welcome(props) {
return <h1>Hello, {props.name}</h1>

}

  • 类组件

    class Welcome extends React.Component {
render() {
return <h1>Hello, {this.props.name}</h1>;
}

}

我们来理解一下上述代码的执行流程（上文也提到过，类组件和函数组件在 React 看来是等价的）：

  1. 我们调用ReactDOM.render()，将element组件当作参数传入；
  2. React 调用 Welcome组件，并传入{name: Mobuo}作为props；
  3. 我们定义的 Welcome组件返回<h1>Hello, Mobuo</h1>元素；
  4. React DOM 更新 DOM来匹配这一描述。

注意：自定义组件一定要以大写字母开头
react 会将小写字母开头的元素是为 HTML 原生标签，如<div>、<h1>等。

如果你想了解更多关于上述规定的本质，可以阅读JSX In Depth.

Composing Components

组件可以在其输出（返回）中引用其他组件，我们可以创建一个名为App的组件，在其输出中多次引用Welcome组件：

function App(props) {
  return (
    <div>
      <Welcome name="Sara"/>
      <Welcome name="Mobuo"/>
      <Welcome name="ChongJiang"/>
  	</div>
  )
}

ReactDOM.render(App, document.getElementById('root'));

一般来说，每个新 React 应用的顶层组件都是App组件，然如果你需要将 React 整合到现有的应用中，则你更可能使用一些类似于Button的小组件，而不是App组件。

Extracting Components

将组件拆分为更小的组件

例如，考虑下面的Comment组件：

function Comment(props) {
  return (
  <div className="Comment">
    <div className="UserInfo">
    <img className="Avatar"
      src={props.author.avatarUrl}
      alt={props.author.name}
    />
    <div className="UserInfo-name">
      {props.author.name}
    </div>
    </div>
    <div className="Comment-text">
    {props.text}
    </div>
    <div className="Comment-date">
    {formatDate(props.date)}
    </div>
  </div>
  );
}

它接受author（Object）、text（string），和date（date）作为参数（props），输出对一个网页评论的描述。

由于存在相互嵌套，因此修改这个组件是有些棘手的，并且很难复用它的各个部分。因此让我们来拆分（Extract）这个组件：

首先，我们提取组件中的Author：

function Author(props) {
  <img className="Avatar"
    src={props.user.avatarUrl}
    alt={props.user.name}
  />
}

Avatar不需要知道评论的内容，因此把参数改成了更一般的名称，：user而不是avatar。这是因为，为了更好的分离与复用，我们推荐从组件本身的角度来命名参数，而不是通过使用时的上下文来命名。

现在我们的的Comment变得稍微简单了：

function Comment(props) {
  return (
  	<div className="Comment">
      <div className="UserInfo">
        <Avatar user={props.author} />
        <div className="UserInfo-name">
        {props.author.name}
        </div>
      </div>
      <div className="Comment-text">
        {props.text}
      </div>
      <div className="Comment-date">
        {formatDate(props.date)}
      </div>
    </div>
  );
}

下一步，我们提取UserInfo组件，它会在渲染Avatar及评用户昵称：

function UserInfo(props) {
  return (
    <div className="UserInfo">
      <Avatar user={props.user}></Avatar>
      <div className="UserInfo-name">
        {props.user.name}
      </div>
    </div>
  )
}

这使我们的Comment组件进一步简化：

function Comment(props) {
  return (
    <div className="Comment">
      <UserInfo user={props.author} />
      <div className="Comment-text">
        {props.text}
      </div>
      <div className="Comment-date">
        {formantDate(props.date)}
      </div>
    </div>
  )
}

提取组件看起来好像是繁重无味的，但对于一个大型应用来说，有一个可复用的调色板是值得的。对于那些在 UI 中经常使用的组件，如Button, Panel, Avatar，或是本身就十分复杂Comment, todo-list，这些都可以作为提取组件的候选。

Props are Read-Only

如果你定义了一个类组件和函数组件，一定不要修改它的props。

考虑如下sum函数：

function sum(a, b) {
  return a + b;
}

这种函数被称为纯函数（pure），因为它不会修改它的输入，并且对于相同的输入，输出总是相同的。

相反，下面这种函数不是纯函数：

function withdraw(account, amount) {
  account.total -= amount;
}

React 是灵活的，但有一个严格的规则：所有的 React 组件一定必须像纯函数一样运行。

当然，UI 是动态的，并且随时都有可能改变，在下一节，我们会介绍一个新概念：状态state，state 可以保证在不触犯上面的规则的修改组件的输出，以响应用户的交互、网络请求等。
