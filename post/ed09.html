动态SQL

在传统的JDBC编程中，拼装并执行SQL语句是一个相当冗杂且烦琐的过程，作为一个ORM框架，MyBatis提供了更加灵活高效的方式：动态SQL，通过动态SQL，开发人员可以根据需要动态组装SQL语句，MyBatis还支持通过注解配置Sql语句

MyBatis提供了一系列动态SQL元素，其中常用的元素如下表所示：

设置参数 描述
if 相当于判断语句，常用于单条件分支判断
choose(when, otherwise) 相当于判断语句，常用于多条件分支判断
trim(where, set) 辅助元素，用于处理一些拼装问题
foreach 相当于循环语句，在in语句等列举条件时常用

接下来对这些参数进行详细的说明

<if>元素

<if>元素主要用于条件判断，它类似于Java代码中的if语句，通常与test属性联合使用。

在实际开发中，<if>元素的任务是根据需求动态控制where子句中的内容，

现有一张学生表，程序可通过学生姓名，年龄，科目等字段查询学生信息，如果将科目作为非必选的查询条件

满足既可以通过姓名和科目查询学生信息，也满足可以直接通过姓名查询学生信息时，需要使用<if>元素。

创建POJO类

Student.java

package io.gitee.yunchongxiao.pojo;
public class Student {
    private int sid;
    private String sname;
    private String age;
    private String course;

    public Student() {
    }

    public Student(String sname, String age, String course) {
        this.sname = sname;
        this.age = age;
        this.course = course;
    }

    public int getSid() {
        return sid;
    }

    public void setSid(int sid) {
        this.sid = sid;
    }

    public String getSname() {
        return sname;
    }

    public void setSname(String sname) {
        this.sname = sname;
    }

    public String getAge() {
        return age;
    }

    public void setAge(String age) {
        this.age = age;
    }

    public String getCourse() {
        return course;
    }

    public void setCourse(String course) {
        this.course = course;
    }

    @Override
    public String toString() {
        return "Student{" +
                "sid=" + sid +
                ", sname='" + sname + '\'' +
                ", age='" + age + '\'' +
                ", course='" + course + '\'' +
                '}';
    }
}

<choose>、<when>、<otherwise>元素

  • <choose>元素和<id>元素类似，但适用于多判断的场景，类似于switch语句

  • <choose>元素通常包含<when>和<otherwise>两个子元素

  • 其中至少包含一个<when>元素、0个或1个<otherwise>元素

现进行如下要求的查询：

  1. 若sid不为空，则通过sid查询学生信息
  2. 若sid为空，sname不为空，则通过sname模糊查询学生信息
  3. 若sname也为空，则根据课程查询学生信息

映射文件

<select id="selectStudents" parameterType="student" resultType="student">
    select * from student where 1 = 1
    <choose>
        <when test="null != sid and '' != sid">
            and sid = #{sid}
        </when>
        <when test="null != sname and '' != sname">
            and sname like '%${sname}%'
        </when>
        <otherwise>
            and course = 'Java'
        </otherwise>
    </choose>
</select>

<where>元素

细心的读者应该注意到了，上述的sql查询语句中含有一个"where 1=1"，这是为了和后面的带有and的条件相连接，但是这么丑的句法片段可以消除掉吗，of course！我们使用<where>元素就可以代替上述的带有where 1=1的choose元素了

  • 注意，这里的的多个if语句是相互独立的，不存在if elif 这种相关关系，所以不会存在第一个if满足则不执行下面的if这种情况。
<select id="selectStudentByWhere" parameterType="student" resultType="student">
    select * from student
    <where>
        <if test="null != sid and '' != sid">
            and sid = #{sid}
        </if>
        <if test="null != sname and '' != sname">
            and sname = '%${name}%'
        </if>
        <if test="null != course and '' != course">
            and course = #{course}
        </if>
    </where>
</select>

<set>元素

<set>元素和<where>元素的用法类似，但是set是使用在update语句中的

如果<set>元素中的子元素含有返回值，则set会在sql语句后相应的追加一个set子句

<update id="updateStudentBySet" parameterType="student">
    update student
    <set>
        <if test="null != sname and '' != sname">
            sname = #{sname},
        </if>
        <if test="null != course and '' != course">
            course = #{course},
        </if>
    </set>
    where sid = #{sid}
</update>

编写测试类

@Test
public void updateStudentBySet() throws IOException {
    SqlSession sqlSession = getSession();
    Student student = new Student();
    student.setSname("Noiris");
    student.setCourse("C++");
    student.setSid(3);
    int result = sqlSession.update("updateStudentBySet", student);
    if (result > 0){
        System.out.println("成功更新" + result + "条数据");
        sqlSession.commit();
    }
    else{
        System.out.println("更新失败");
    }
    sqlSession.close();
}

成功更新1条数据

<trim>元素

<trim>元素用于删除多余的关键字，它可以直接实现where和set元素的功能

<trim>包含四个属性：

设置参数 描述
prefix 指定给SQL语句增加的前缀
prefixOverrides 指定SQL语句中要去掉的前缀字符串
suffix 指定给SQL语句添加的后缀
suffixOverides 指定SQL语句中要去掉的后缀字符串

实现上述的select语句

<select id="selectStudentByTrim" parameterType="student" resultType="student">
    select * from student
    <trim prefix="where" prefixOverrides="and">
        <if test="null != sid and '' != sid">
            and sid = #{sid}
        </if>
        <if test="null != sname and '' != sname">
            and sname like '%${sname}%'
        </if>
        <if test="null != course and '' != course">
            and course = #{course}
        </if>
    </trim>
</select>

实现上述的update语句

<update id="updateStudentByTrim" parameterType="student">
    update student
    <trim prefix="set" suffixOverrides=",">
        <if test="null != sname and '' != sname">
            sname = #{sname},
        </if>
        <if test="null != course and '' != course">
            sname = #{sname},
        </if>
    </trim>
    where sid = #{sid}
</update>

<foreach>元素

foreach主要用来遍历，能够支持数据、List或Set接口的集合。主要包含五个属性：

设置参数 描述
collection 指定传递进来的参数名称，可以代表List、Set等集合
item 指定循环中当前的元素
index 指定当前元素在集合的位置下标
open、close 指定包装集合元素的符号
separator 指定各元素的间隔符

foreach通常用于SQL语句中的in关键字：

<select id="selectStudentByForeach" resultType="student">
    select * from student where sid in
    <foreach collection="list" index="index" item="sid" open="(" separator="," close=")" >
        #{sid}
    </foreach>
</select>

编写测试类

@Test
public void selectStudentByForeach() throws IOException {
    SqlSession sqlSession = getSession();
    ArrayList<Integer> idList = new ArrayList<Integer>();
    idList.add(1);
    idList.add(2);
    idList.add(3);

    List<Student> studentList = sqlSession.selectList("selectStudentByForeach", idList);
    for (Student stu : studentList) {
        System.out.println(stu);
    }
    sqlSession.close();
}

运行结果

Student{sid=1, sname='ZhangSan', age='20', course='Java'}
Student{sid=2, sname='LiSi', age='21', course='Java'}
Student{sid=3, sname='Noiris', age='20', course='C++'}

<bind>元素

在实际开发中，由于不同的数据库支持的SQL语法略有不同，若需要更换数据库类型，则可以通过<bind>来消除差异

<bind>元素把OGNL表达式的值绑定到一个变量上，通过<bind>元素，能够简化对变量的值的引用

<select id="selectStudentByBind" parameterType="student" resultType="student">
    select * from student where
    <bind name="sname_pattern" value="'%' + sname + '%'"/>
    <if test="null != sname_pattern and '' != sname_pattern">
        sname like #{sname_pattern}
    </if>
</select>

编写测试类

@Test
public void selectStudentByBind() throws IOException {
    SqlSession sqlSession = getSession();
    Student student = new Student();
    student.setSname("Li");
    List<Student> studentList = sqlSession.selectList("selectStudentByBind", student);
    for (Student stu : studentList) {
        System.out.println(stu);
    }
    sqlSession.close();
}

运行结果

Student{sid=2, sname='LiSi', age='21', course='Java'}

注解

在MyBatis中，除了使用xml映射的方式，还支持通过注解实现POJO对象和数据库表之间的映射。使用注解时，一般将Sql语句直接写在接口上。注解使用简单，且不会造成大量开销。常用的注解如下表所示：

注解 描述
@Select 查询操作
@Insert 插入操作
@Update 更新操作
@Delete 删除操作
@Param 传入参数

@Select注解

@Select注解用于映射查询语句，其作用相当于xml文件中的<select>元素，接下来让我们看一看代码吧

创建Mapper接口

StudentMapper.java

package io.gitee.yunchongxiao.mapper;

import io.gitee.yunchongxiao.pojo.Student;
import org.apache.ibatis.annotations.Select;

public interface StudentMapper {
    @Select("select * from student where sid = #{sid}")
    Student selectStudentBySid(int sid);
}

关联接口文件

<mapper class="io.gitee.yunchongxiao.mapper.StudentMapper"/>

编写测试类

@Test
public void selectStudentBySid() throws IOException {
    SqlSession sqlSession = getSession();
    StudentMapper mapper = sqlSession.getMapper(StudentMapper.class);
    Student student = mapper.selectStudentBySid(3);
    System.out.println(student);
    sqlSession.close();
}

运行结果

Student{sid=3, sname='Noiris', age='20', course='C++'}
