AOP基础

AOP简介

AOP（Aspect Oriented Programming，面向切面编程），和OOP不同，它主张将程序中的相同业务逻辑进行横向隔离，并将重复的业务逻辑抽取到一个独立的模块中，最终实现提升程序可复用性和开发效率的目的

AOP的基本术语

连接点（Joinpoint)

程序执行中某个特定的节点，如某个类初始化后、某个方法执行前、程序处理异常时等等。

只要是具有边界性质的特定点，即与方法执行相关的特定节点，都可以作为连接点。

通知（Advice）

通知是在目标类连接点上执行的一段代码，包括around、before、after等不同类型。

在Spring中，它主要描述围绕方法调用而注入的行为，具体如下表所示：

通知类型 描述
前置通知（Before） 在目标方法被调用之前通知
后置通知（After） 在目标方法被调用之后通知
返回通知（After-returning） 在目标方法成功执行之后通知
异常通知（After-throwing） 在目标方法抛出异常之后通知
环绕通知（Around） 在目标方法被调用的前后进行通知

切点（Pointcut）

匹配连接点的断言，AOP通过切点来定位特定的连接点，通知和一个切入点表达式关联，并在满足这个切入点的连接点上运行

切入点表达式如何和连接点匹配是AOP的核心。

目标对象（Target）

通知所作用的目标类，如果缺少AOP的支持，目标业务类就要独立完成所有业务逻辑，为了降低冗余，可以目标业务类可以借助AOP将重复的代码抽取出来。

引介（Introduction）

引介是一种特殊的通知，它为类添加一些属性和方法，来实现一个类原本没有的接口

切面（Aspect）

对系统中的横切关注点逻辑进行模块化封装的AOP概念实体，关注点模块化之后，可能会横切多个对象，

SpringAOP是实施切面的具体方法，它将切面所定义的横切逻辑添加到切面指定的连接点中。

织入（Weaving）

织入是将通知添加到目标类具体连接点的过程，Spring采用动态代理织入，而AspectJ采用编译期织入和类加载器织入

代理（Proxy）

目标类被AOP织入增强后产生的一个结果类，这个类融合了原类和增强的逻辑，根据不同的代理方式，代理类既可能是和原类具有相同接口的类，也可能是原类的子类，不管怎样，都可以采用与调用原类相同的方法调用代理类。

SpringAOP的实现机制

JDK动态代理

在Spring中，AOP代理由Spring的IOC容器负责创建，其依赖关系也由IOC容器负责管理，因此SpringAOP可以直接将IOC容器中的其他Bean对象实例作为目标对象，在默认状态下，SpringAOP采用JDK动态代理，当目标对象是一个类且这个类没有实现接口时，Spring会切换使用CGLib代理

JDK动态代理主要涉及两个API：InvocationHandler和Proxy，它们位于java.lang.reflect包中

其中InvocationHandler是一个接口，代理类可以通过实现该接口定义横切逻辑，并将横切逻辑和业务逻辑编织在一起；

Proxy利用InvocationHandler动态生成目标类的代理对象。

创建JDK动态代理

PerformHandler.java

package io.gitee.yunchongxiao.demo01;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;

public class PerformHandler implements InvocationHandler {
    private Object target;

    public PerformHandler(Object target) {
        this.target = target;
    }

    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("方法开始执行");
        Object invoke = method.invoke(target, args);
        System.out.println("方法执行完毕");
        return invoke;

    }
}

CGLib动态代理

JDK动态代理存在缺陷，它只能为接口创建代理实类，当需要为类创建代理实类时，要使用CGLib动态代理

CGLib动态代理不要求目标对象实现接口，它采用底层的字节码，通过继承的方式动态创建代理对象

CGLib动态代理依靠Enhancer类创建代理实例，依靠MethodInterceptor接口织入要增强的方法

创建CGLib动态代理

CglibProxy.Java

package io.gitee.yunchongxiao.demo01;

import java.lang.reflect.Method;
import org.springframework.cglib.proxy.*;

public class CglibProxy implements MethodInterceptor {
    private Enhancer enhancer = new Enhancer();

    public Object getProxy(Class clazz) {
        enhancer.setSuperclass(clazz);
        enhancer.setCallback(this);
        return enhancer.create();
    }

    @Override
    public Object intercept(Object obj, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {
        System.out.println("CGLib代理之前");
        Object invoke = methodProxy.invokeSuper(obj, objects);
        System.out.println("CGLib代理之后");
        return invoke;
    }
}

创建测试类

@Test
public void cglibProxy() {
    CglibProxy cglibProxy = new CglibProxy();
    LoginServiceImpl userService = (LoginServiceImpl) cglibProxy.getProxy(LoginServiceImpl.class);
    userService.login();
}

编写代理类

AnnoAdvice.Java

package io.gitee.yunchongxiao.demo02;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;

@Aspect
public class AnnoAdvice {
    @Pointcut("execution(* io.gitee.yunchongxiao.demo02.UserServiceImpl.*(..))")
    public void pointcut() {
    }

    @Before("pointcut()")
    public void before() {
        System.out.println("这是前置通知");
    }

    @After("pointcut()")
    public void after() {
        System.out.println("这是后置通知");
    }

    @AfterReturning("pointcut()")
    public void afterReturning() {
        System.out.println("这是后置通知（方法不抛出异常时调用）");
    }

    @Around("pointcut()")
    public Object around(ProceedingJoinPoint point) throws Throwable {
        System.out.println("这是环绕通知之前的部分");
        Object object = point.proceed();
        System.out.println("这是环绕通知之后的部分");
        return object;

    }

    @AfterThrowing("pointcut()")
    public void afterThrowing() {
        System.out.println("这是异常通知");
    }
}

多个切面的优先级

实际开发中可能需要在多个切面中定义多个通知织入同一个切点，这就要考虑多个切面的优先级；

切面的优先级有三种配置方式：配置@Order注解、xml文件、切面类实现Ordered接口

优先级越高的切面，他的通知方法优先执行，且最后退出。

性能监控

性能监控是一个成熟的系统必须具备的功能，他主要通过监控运行状况并识别瓶颈来提高应用程序的性能。

一般来说，系统中重要的模块都要编写监控性能的代码，这里特别适合使用AOP实现性能监控。

package io.gitee.yunchongxiao.demo03;

import java.util.Date;

public class Record {
    private String className;
    private String methodName;
    private Date recordTime;
    private Long expendTime;

    public String getClassName() {
        return className;
    }

    public void setClassName(String className) {
        this.className = className;
    }

    public String getMethodName() {
        return methodName;
    }

    public void setMethodName(String methodName) {
        this.methodName = methodName;
    }

    public Date getRecordTime() {
        return recordTime;
    }

    public void setRecordTime(Date recordTime) {
        this.recordTime = recordTime;
    }

    public Long getExpendTime() {
        return expendTime;
    }

    public void setExpendTime(long expendTime) {
        this.expendTime = expendTime;
    }

    @Override
    public String toString() {
        return "Record{" +
                "\n\tclassName='" + className + '\'' +
                ", \n\tmethodName='" + methodName + '\'' +
                ", \n\trecordTime='" + recordTime + '\'' +
                ", \n\texpendTime='" + expendTime + '\'' +
                "\n}";
    }
}

RecordAspect.Java

package io.gitee.yunchongxiao.demo03;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Pointcut;

import java.util.Date;

@Aspect
public class RecordAspect {
    @Pointcut("execution(* io.gitee.yunchongxiao.demo03.Service.*(..))")
    public void record() {
    }

    @Around("record()")
    public Object recordTimer(ProceedingJoinPoint pointcut) throws Throwable {
        String className = pointcut.getTarget().getClass().getName();
        String methodName = pointcut.getSignature().getName();
        long startTime = System.currentTimeMillis();
        Object result = pointcut.proceed();
        long endTime = System.currentTimeMillis();
        Record record = new Record();
        record.setClassName(className);
        record.setExpendTime(endTime - startTime);
        record.setMethodName(methodName);
        record.setRecordTime(new Date());
        System.out.println(record);
        return result;

    }
}

异常监控

在软件系统中，处理异常的策略及成效也是一项重要指标

异常监控和业务逻辑基本不会耦合在一起，同城会抽取出来作为一项独立的模块。

因此Spring的AOP非常适合进行异常监控

编写抛出异常的类

package io.gitee.yunchongxiao.demo03;

public class Service02 {
    public void service() throws Exception {
        System.out.println("执行service()方法");
        int num = 105;
        if (num > 100 || num < 0) {
            throw new MyException("您输入的不正确，请输入0~100之间的数字");
        } else {
            System.out.println("您输入的数字为" + num);
        }
    }
}

编写信息记录类

package io.gitee.yunchongxiao.demo03;

import java.util.Date;

public class Message {
    private String className;
    private String methodName;
    private Date recordTime;
    private String exceptionMsg;

    @Override
    public String toString() {
        return "Message{" +
                "\n\tclassName='" + className + '\'' +
                ", \n\tmethodName='" + methodName + '\'' +
                ", \n\trecordTime=" + recordTime +
                ", \n\texceptionMsg='" + exceptionMsg + '\'' +
                "\n}";
    }

    public String getClassName() {
        return className;
    }

    public void setClassName(String className) {
        this.className = className;
    }

    public String getMethodName() {
        return methodName;
    }

    public void setMethodName(String methodName) {
        this.methodName = methodName;
    }

    public Date getRecordTime() {
        return recordTime;
    }

    public void setRecordTime(Date recordTime) {
        this.recordTime = recordTime;
    }

    public String getExceptionMsg() {
        return exceptionMsg;
    }

    public void setExceptionMsg(String exceptionMsg) {
        this.exceptionMsg = exceptionMsg;
    }
}

编写异常监控切面类

package io.gitee.yunchongxiao.demo03;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Pointcut;

import java.util.Date;

@Aspect
public class MessageAspect {

    @Pointcut("execution(* io.gitee.yunchongxiao.demo03.Service02*.*(..))")
    public void exceptionMsg() {
    }

    @Around("exceptionMsg()")
    public Object msgMethod(ProceedingJoinPoint joinPoint) throws Throwable {
        String className = joinPoint.getTarget().getClass().getName();
        String methodName = joinPoint.getSignature().getName();
        try {
            return joinPoint.proceed();
        } catch (MyException myException) {
            Message msg = new Message();
            msg.setClassName(className);
            msg.setMethodName(methodName);
            msg.setRecordTime(new Date());
            msg.setExceptionMsg(myException.getMsg());
            System.out.println(msg);
            return null;
        }
    }

}
