第三章-MyBatis关系映射

发表于2021-01-03|更新于2022-01-31|JavaEE（ssm）企业应用实战
|字数总计:5.1k|阅读时长:26分钟|阅读量:|评论数:

在实际开发中，往往标语表之间存在着某种关联关系，如一对一、一对多、多对多等，MyBatis该如歌操作这些关联的表呢，下面将会根据这个问题进行详细展开。

表与表之间的关系

表与表之间的关系一般为一对一、一对多、多对多等，这里的对应关系是指表的数据之间的对应关系

  • 一对一：A表的一条数据与B表的一条数据对应、如学生表中的一个学生信息和校园卡表的一个校园卡信息对应；
  • 一对多：A表的一条数据与B表的多条数据对应，如班级表中的一个班级信息和学生表中的属于该班级的所有学生信息对应；
  • 多对多：A表的多条数据与B表的多条数据对应，如学生表中的多个学生信息和教师表中的多个教师信息对应。

如果直接使用sql语句维护数据表，则通常在任意一方引入对方的主键作为外键，但是在企业开发中，我们常常使用POJO类

MyBatis提供了<resultMap>元素来实现上述的关联操作。

一对一

  • MyBatis通过<association>元素来进行一对一引用

  • 使用resultMap作为返回类型，表示通过id引用下述的结果集

  • 结果集中的column属性用来指定对应的表字段，property属性用来指定POJO中实体类的成员

  • 在连接表的时候使用association字段进行连接，property表示POJO类的实体类成员，JavaType表示该成员的Java类型

对于下面的学生信息表和学生卡信息表，查询一个学生及其拥有的校园卡信息（下方是期望的查询结果）：

Stu{sid=1, sname='ZhangSan', age='20', course='Java', sc=StuCard{cid=0, balance=1000.5}}

创建映射文件

StuMapper.xml

一张表的一条数据和令一张表的一条数据关联查询，我们使用<resultMap>元素的<association>子元素进行处理，其配置代码如下：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!-- 映射命名空间 -->

<mapper namespace=“stu”>

<!-- 选择语句，其中resultMap说明返回值是一个结果集处理的引用？ -->

<select id=“selectStuBySid” parameterType=“int” resultMap=“stuResultMap”>

<!-- 正常写选择语句即可 -->

select s.*, c.balance from stu s, stu_card as c where s.cardid = c.cid and s.sid = #{sid};

</select>

<!-- 这里的id唯一标识该结果集，该id在上条选择语句的resultMap属性被引用了 -->

<resultMap id=“stuResultMap” type=“stu”>

<!-- 这里的id表示该表的主键，column表示表的字段名，property表示POJO类的成员名称 -->

<!-- 这里只要在创建POJO类时保持和表中字段名一致就没什么问题了 -->

<!-- 下面的每一个id/result都对应上述查询语句选择出的一个字段 -->

<id column=“sid” property=“sid”/>

<result column=“sname” property=“sname”/>

<result column=“age” property=“age”/>

<result column=“course” property=“course”/>

<!-- 使用association进行表与表之间的连接处理，这里显式设置了JavaType，可能是由于非基本类型无法自动识别？ -->

<association property=“sc” javaType=“stuCard”>

<!-- 用id标识表的主键字段 -->

<id column=“cid” property=“cid”/>

<!-- 用result标识表的非主键字段 -->

<result column=“balance” property=“balance”/>

</association>

</resultMap>

</mapper>

  • 使用resultMap作为返回类型，表示通过id引用下述的结果集

  • 结果集中的column属性用来指定对应的表字段，property属性用来指定POJO中实体类的成员

  • 在连接表的时候使用collection元素进行连接，property表示POJO类的实体类成员，JavaType表示该成员的Java类型

StuClassMapper.xml

我们使用<collection>元素类实现，配置代码如下：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!-- 命名空间 -->

<mapper namespace=“stuClass”>

<!-- 选择元素，这里返回类型还是使用resultMap表示结果集的引用 -->

<select id=“selectStuClassByCid” parameterType=“int” resultMap=“stuClassResultMap”>

select c.*, s.sid, s.sname, s.course from stu_info s, stu_class c

where s.classid = c.cid and c.cid = #{cid};

</select>

<!-- 这里编配置结果集需要执行的操作 -->

<resultMap id=“stuClassResultMap” type=“stuClass”>

<!-- id标识表的主键字段 -->

<id column=“cid” property=“cid”/>

<!-- result标识表的非主键字段 -->

<result column=“cname” property=“cname”/>

<result column=“sum” property=“sum”/>

<id column=“sid” property=“sid”/>

<result column=“sname” property=“sname”/>

<result column=“age” property=“age”/>

</collection>

</resultMap>

</mapper>

主键映射

当数据库中的表的主键被设置为自增主键时（上述举例的表都是如此），我们在插入信息时就不需要设置主键信息

若我们想在插入后立即获取主键值，则可以使用<useGeneratedKeys>实现：

编写配置文件

<insert id="insertStuClass" parameterType="stuClass" useGeneratedKeys="true" keyProperty="cid">
    insert into stu_class(cname, sum) values (#{cname}, #{sum});
</insert>

@Test
public void insertStuClass() {
    String resource = "mybatis-config.xml";
    try {
        InputStream inputStream = Resources.getResourceAsStream(resource);
        SqlSessionFactory sessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        SqlSession sqlSession = sessionFactory.openSession();
        StuClass stuClass = new StuClass();
        stuClass.setCname("Java1803");
        stuClass.setSum(50);
        int result = sqlSession.insert("insertStuClass", stuClass);
        if (result > 0) {
            System.out.println("成功插入" + result + "条消息");
            System.out.println("被插入数据的主键为: " + stuClass.getCid());
            sqlSession.commit();
        } else {
            System.out.println("插入失败");
            sqlSession.rollback();
        }
        sqlSession.close();
    } catch (IOException e) {
        e.printStackTrace();
    }
}
