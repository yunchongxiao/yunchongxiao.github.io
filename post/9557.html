IOC和DI

简介

IOC（Inversion Of Control，控制反转），它是一种设计思想，是指将对象的控制权由程序代码反转给外部容器。

Spring容器会帮助开发人员进行类之间的依赖关系和生命周期管理

DI（Dependency Injection，依赖注入），他是控制反转的另一种说法，同时也为控制反转提供了实现方法

依赖注入是指调用类对其它类的依赖关系由容器注入，这就避免了调用类对其他类的过度依赖， 降低了类一类之间的耦合

依赖注入的方式

构造器注入、属性注入、接口注入，其中构造器注入和属性注入是常用的两种方法，下面就详细的介绍一下

Bean的配置

Bean的定义

<bean>元素提供了许多属性用于描述bean的信息，下面列出几个常用的属性：

属性 描述
Class 指定Bean对应类的全路径
Name 指定Bean对应对象的一个标识
Scope 指定Bean对象的作用域
id Bean对象的唯一标识
lazy-init 是否延迟加载，默认为false
init-method 对象初始化方法
destroy 对象销毁方法

注入集合

在实际开发中，有时需要将集合注入到Bean中，如List、Set、Map、Array等，下面介绍以下这些集合的注入方法

Mix.java

package io.gitee.yunchongxiao.bean;

import java.util.Arrays;
import java.util.List;
import java.util.Map;
import java.util.Set;

public class Mix {
    private List<String>  list;
    private Map<String, String> map;
    private Set<String> set;
    private String[] array;

    public void setList(List<String> list) {
        this.list = list;
    }

    public void setMap(Map<String, String> map) {
        this.map = map;
    }

    public void setSet(Set<String> set) {
        this.set = set;
    }

    public void setArray(String[] array) {
        this.array = array;
    }

    @Override
    public String toString() {
        return "Mix{" +
                "list=" + list +
                ", map=" + map +
                ", set=" + set +
                ", array=" + Arrays.toString(array) +
                '}';
    }
}

使用P: 命名空间注入

前面说过，Spring的配置文件一般基于<property>元素来配置Bean的属性，但是若Bean实例的属性足够多时，使用大量的property就显得冗余，这时间，可以使用P: 命名空间

p: 命名空间的实现需要在配置文件中添加：

xmlns:p="http://www.springframework.org/schema/p"

例如上述的School对象，可以使用以下方式：

<bean id="pSchool" class="io.gitee.yunchongxiao.bean.School" p:sid="1" p:student-ref="student"/>

使用SpEL注入

SpEL是Spring Expression Launguage（Spring表达式语言）的缩写，它是一种支持查询和操作运行时对象导航图功能的表达式语言，可以简化开发，能够减少代码逻辑和配置信息的编写。它以#{…}为定界符，能够为Bean的属性进行过动态赋值。

applicationContext.xml

<bean id="employee" class="io.gitee.yunchongxiao.bean.Employee">
    <property name="eid" value="1"/>
    <property name="department" value="研发部"/>
    <property name="name" value="Noiris"/>
</bean>

<bean id="person" class="io.gitee.yunchongxiao.bean.Person" >
    <property name="pid" value="#{employee.eid}"/>
    <property name="name" value="#{employee.name}"/>
</bean>

TestSpring.java

@Test

Bean的作用域

当Spring容器创建一个Bean实例时，不仅可以完成Bean的实例化，还可以为该Bean指定作用域，这个作用域是指不同Bean实类之间的请求可见范围。具体由五种作用域，如下表所示：

属性 描述
Singleton（默认） 单例模式，在Spring容器中只会存在一个共享的实例
Prototype 每次从容器中调用Bean时，都会产生一个新的Bean实例
Request 一个HTTP请求会产生一个Bean对象，仅用于基于Web的Spring
Session 限定一个Bean的作用域为HTTPsession的生命中周期，仅用于基于Web的Spring
global Session 限定一个Bean的作用域为全局HTTPsession的生命周期，仅用于基于Web的Spring

下面一起研究一下Singleton和prototype吧

修改配置文件

<bean id="person01" class="io.gitee.yunchongxiao.bean.Person"/>
<bean id="person02" class="io.gitee.yunchongxiao.bean.Person" scope="prototype"/>

@Test
public void personScope() {
    ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
    Person person01_1 = applicationContext.getBean("person01", Person.class);
    person01_1.setName("singleton");
    Person person01_2 = applicationContext.getBean("person01", Person.class);
    Person person02_1 = applicationContext.getBean("person02", Person.class);
    person02_1.setName("prototype");
    Person person02_2 = applicationContext.getBean("person02", Person.class);
    System.out.println(person01_1);
    System.out.println(person01_2);
    System.out.println(person02_1);
    System.out.println(person02_2);
    System.out.println(person01_1 == person01_2);
    System.out.println(person02_1 == person02_2);
}

Bean的生命周期

Bean的生命周期是指Bean实例被创建、初始化和销毁的过程， 这里需要注意的两个时间点是Bean实例被创建的时候和被销毁的时候。Spring为这两个时间点提供了接口，分别是init-method和destroy-method属性或者实现InitializationBean接口和disposableBean接口。

使用属性方法

这种方法要求类成员中存在初始化函数和销毁函数（我这里命名为init()和destroy()）

配置文件

<bean id="beanMethod" class="io.gitee.yunchongxiao.bean.BeanMethod" init-method="init" destroy-method="destroy"/>

测试类

@Test
public void getBeanByMethod() {
    ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
    BeanMethod beanMethod = applicationContext.getBean("beanMethod", BeanMethod.class);
    System.out.println(beanMethod);
    // 关闭容器的操作
    AbstractApplicationContext ac = (AbstractApplicationContext)applicationContext;
    ac.registerShutdownHook();
}

运行结果

Bean的初始化完成，init()方法已调用
Bean01{bid=0, bname='null'}
Bean的销毁已完成，destroy()方法已调用

实现接口

package io.gitee.yunchongxiao.bean;
import org.springframework.beans.factory.InitializingBean;
import org.springframework.beans.factory.DisposableBean;
public class BeanApi implements InitializingBean, DisposableBean{
    public int bid;
    public String name;

    public int getBid() {
        return bid;
    }

    public void setBid(int bid) {
        this.bid = bid;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "BeanApi{" +
                "bid=" + bid +
                ", name='" + name + '\'' +
                '}';
    }

    @Override
    public void afterPropertiesSet() {
        System.out.println("Bean的初始化完成，InitializingBean接口已实现");
    }

    @Override
    public void destroy() {
       System.out.println("Bean的销毁已完成，DisPosableBean接口已实现");
   }
}

注解

注解可以直接写在代码中，减少了xml文件的配置内容，且注解能够实现自动装配，提供的功能也更强大

常用的注解如下图所示：

注解 描述
@Component 指定一个普通的Bean
@Controller 指定一个逻辑控制组件Bean，功能上等同于@Component
@Service 指定一个业务逻辑组件Bean，功能上等同于@Component
@Repository 指定一个DAO组件Bean，功能上等同于@Component
@Scope 指定Bean实例的作用域
@Value 指定Bean实例的注入值
@Autowired 指定要自动装配的对象
@Qualifier 指定要自动装配的对象名称，通常与@Autowired联合使用
@resource 指定要注入的对象
@PostConstruct 指定Bean实例完成初始化后调用的方法
@PreDestroy 指定Bean实例销毁之前调用的方法

package io.gitee.yunchongxiao.bean;

                "sid=" + sid +
                ", stu=" + stu +
                '}';
    }

    public void init() {
        System.out.println("Bean的初始化完成，init()方法已调用");
    }

    public void destroy(){
        System.out.println("Bean的销毁已完成，destroy()方法已调用");
    }
}
