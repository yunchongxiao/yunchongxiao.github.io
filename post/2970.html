SpringMVC概述

在Java EE企业级开发中，MVC是构建Web程序的最流行的框架。MVC中的M表示Model（模型）、V表示View（视图）、C表示Controller（控制器）

通常，M由JavaBean充当、V由JSP充当，C由Servlet充当。

SpringMVC简介

SpringMVC是Spring的一个模块，同时也是一个可用于构建Web程序的MVC框架

SpringMVC提供了一个前端控制器来分发请求，同时，它还支持包括JSP、FreeMarker、Velocity等在内的多项试图技术

SpringMVC灵活、高效、配置方便，与其他MVC框架相比，它可以与Spring无缝集成并使用Spring的功能，表现出更好的复用性和扩展性。

SpringMVC的功能组件

Spring通过一系列组件实现功能，这下组件包括：

  1. DispatcherServlet（前端控制器）

    前端控制器负责拦截客户端请求并分发给其他组件，它是整个流程控制的中心，负责调度其他组件的执行，降低各组件之间的耦合，提升整体效率。前端控制器由框架提供，在程序运行过程中自动实现功能。

  2. HandlerMappering（处理器映射器）

    处理器映射器负责根据客户端请求的URL寻找处理器，SpringMVC提供了配置文件、注解等映射方式，这些映射方式将由相应的处理器映射器负责处理，处理器映射器由框架提供，在程序运行过程中自动实现功能。

  3. Handler（处理器）

    处理器负责对客户端的请求进行处理，由于处理请求设计具体的业务逻辑，因此，开发者需要在处理器中编写处理业务逻辑的代码。

  4. HandlerAdapter（处理器适配器）

    处理器适配器负责根据特定的规则对处理器进行执行，它可以执行多种类型的处理器，是设计模式种适配器模式的具体应用。处理器适配器由框架提供，在程序运行过程中自动实现功能。

  5. ViewResolver（视图解析器）

    视图解析器负责视图解析，它可以处理结果生成View并展示给用户。视图解析器由框架提供，在程序运行过程中自动实现功能。但是View需要由开发人员根据具体需求编写。

SpringMVC的工作流程

通常情况下，SpringMVC在处理客户端请求时要完成以下步骤：

  1. 客户端发出一个HTTP请求，Web应用服务器接收这个请求，如果Web应用的配置文件中指定由DispatcherServlet的映射路径，那么服务器将该请求交给DispatcherServlet处理。
  2. DispatcherServlet接收到请求后，将根据包括URL、方法、报文头和参数在内的请求信息以及HandlerMapping的配置解析出目标Handler。
  3. 在解析出目标Handler后，DispatcherServlet将通过相应的HandlerAdapter来调用Handler并完成业务逻辑的处理。
  4. 在完成业务逻辑处理后，代表处理结果的ModelAndView会被返回到DispatcherServlet，ModelAndView中包含逻辑视图名和模型数据信息。
  5. DispatcherServlet通过ViewResolver完成逻辑视图名到真实View对象的解析。
  6. 获得真实的View对象后，DispatcherServlet将模型数据传给View对象并通过View对象对模型数据进行视图渲染。
  7. DispatcherServlet将最终的View对象响应给客户端并展示给用户。

以上7个步骤时SpringMVC在处理客户端请求时的基本流程，这些步骤中的大部分是由SpringMVC自动完成的，由于Handler需要处理具体业务逻辑，View需要向用户展示页面，因此，开发者只需编写Handler和View相关的内容。

SpringMVC的重要API

DispatcherServlet类

DispatcherServlet是前端控制器，是整个流程控制的中心，它是SpringMVC程序的入口。

DispatcherServlet首先是一个Servlet类，所以，首先要在web.xml中配置DispatcherServlet

web.xml

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">

    <servlet>
        <servlet-name>SpringMVC</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:springmvc-config.xml</param-value>
        </init-param>
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>SpringMVC</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>
</web-app>

DispatcherServlet类的辅助API

为了帮助DispatcherServlet类实现功能，SpringMVC提供了一些辅助API，这些辅助API是和SpringMVC中的组件相对应的，具体如下表所示：

名称 描述
Controller 接口，封装了处理客户端请求的方法，开发者可以实现该接口的类来编写具体业务逻辑的代码
HandlerMapping 用于实现请求处理器的映射，当映射成功时，返回HandlerExecutionChain对象，该对象封装了一个Handler对象，多个HandlerInterceptor对象
HandlerAdapter 适配器设计模式的具体应用，支持多种类型的处理器，调用处理器的handlerRequest()方法进行功能处理
HandlerExceptionResolver 用于处理器异常解析，可将异常提醒转至统一的错误界面
ViewResolver 用于将逻辑视图名解析为具体的View对象
LocaleResolver&LocaleContextResolver 用于解析客户端使用的地区、时区信息，然后具体提供视图国家化的支持
MultipartResolver 用于处理multi-part请求，例如，文件上传等
ModelAndView 用于封装逻辑视图名和模型数据信息

Controller接口

Controller接口位于org.springframework.web.servlet.mvc中，它提供了一个handlerRequest()方法。

开发者在编写Controller的实现类时，可以在HandlerRequest()方法中添加处理业务逻辑的代码。

在SpringMVC工作流程中，DispatcherServlet类会间接调用Controller实现类中的handlerRequest()方法并完成业务处理

最终handlerRequest()方法将以ModelAndView的形式返回结果。

ModelAndView类

ModelAndView类常用于封装Controller的处理结果，在ModelAndView类提供的方法中，有一些是在开发中经常用到的，如下表所示：

方法 描述
ModelAndView 默认的构造方法
ModelAndView(String viewName) 需要传入View名称的构造函数
ModelAndView addObject(Object attributeValue) 向模型添加属性
ModelAndView addObject(String attributeName, Object attributeValue) 通过name-value的形式向模型中添加属性
voidsetView(View view) 设置View对象
voidsetViewName(String viewName) 设置View名称
Boolean hasView() 返回ModelAndView是否具有View名称或View对象

MyController.Java

注意，这里面的modelAndView.setViewName(“page01”)的参数为page01，该参数的完整形式为"/WEB-INF/page/page01.jsp"

由于上一步设置了前缀和后缀，因此这里只需也只能使用page01来进行设置。

package io.gitee.yunchongxiao.controller;
import org.springframework.web.servlet.ModelAndView;
import org.springframework.web.servlet.mvc.Controller;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class MyController01 implements Controller {

    public ModelAndView handleRequest(HttpServletRequest request, HttpServletResponse response) throws Exception {
        ModelAndView modelAndView = new ModelAndView();
        modelAndView.addObject("msg", "Hello World");
        modelAndView.setViewName("page01");
        return modelAndView;
    }
}

SpringMVC的常用注解

为了简化开发，SpringMVC，常用的有：

  1. @Controller，该注解表示一个Controller类。
  2. @RequestMapping，该注解表示Controller类对应请求的路径映射。
  3. @RequestParam
  4. @PathVaribale
  5. @CookieValue
  6. @RequestHeader

@Controller&@RequestMapping

@Controller
publicclassMyController02 {
    @RequestMapping(value="/execute")
    public ModelAndView execute() {
        ModelAndView modelAndView = new ModelAndView();
    	modelAndView.addObject("msg", "Hello Controller&RequestMapping");
    	modelAndView.setViewName("page01");
        return modelAndView;
    }
}
属性 描述
value 指定请求的实际地址
method 指定该方法可以处理的HTTP请求方式，如果没有指定method属性值，则默认映射所有的HTTP请求方式
consumes 指定处理请求的提交内容类型
produces 指定返回的内容类型，返回的内容类型必须是请求头中所包含的类型
params 指定请求中必须包含某些参数，才让该方法处理
headers 指定请求中必须包含某些特性的header指，才让该方法处理
name 为映射的地址指定别名
