Introducing React Hooks

自从React 16.8以来， React Hooks被提出，这可以显著提升我们的编码速度和应用的执行效率，React 可以仅使用函数组件来完成所有功能（这并不代表类组件被弃用了），React Hooks是向后兼容的。

本章，我们主要讨论：

  • React Hooks，并且知道如何使用它们
  • Hooks的规则
  • 如何将类组件迁移到Hooks
  • 根据Hooks和effects理解组件的生命周期
  • 如何从 Hooks获取数据
  • 如何记忆 components, values, and functions with memo , useMemo , and useCallback
  • 如何实现 useReducer

No breaking changes

虽然提出了新的React Hooks，但目前还没有从React中移除类组件的计划。React Hooks只是为React的概念提供了更直接的API，比如props, state, contexts, ref, life cycle等等已知知识。

Using the State Hook

在类组件中，我们使用this.state来使用状态，现在，你可以使用useState钩子

首先，你需要从React中提取useState钩子：

import { useState } from 'react'

然后你需要定义这个状态的state和setter：

const Counter = () => {
  // counter为state，setCounter为setter
  const [counter, setCounter] = useState<number>(0)
}

我们用setCounter作为setter，指定我们只接受number类型，并且将初始值设为0。

为了测试我们的状态，我们需要创建一个由onClick事件触发的方法handleCounter:

const Counter = () => {
  //  在这里使用 useState Hook
  const [counter, setCounter] = useState<number>(0)
  const handleCounter = (operation: string) => {
    return (
      operation === ' add'
      ? setCounter(counter + 1)  // 使用 setter 修改状态
      : setCounter(counter - 1)   // 使用 setter 修改状态
    )
  }
}
return (
  <p>
    Counter: {counter} <br />
    <button onClick={() => handleCounter(' add' ) }>+ Add</button>
    <button onClick={() => handleCounter(' subtract' ) }>- Subtract</button>
  </p>
)

Rules of Hooks

React Hooks基本上是JavaScript函数，但你需要遵循两条规则才能使用它们。React提供了一个linter插件来执行这些规则，你可以通过运行下面的命令来安装:

npm install --save-dev eslint-plugin-react-hooks

让我们看看这两条规则。

Rule 1: Only call Hooks at the top level

不要在循环、条件或嵌套函数中调用Hooks。相反,总是使用挂钩在React函数的顶层。这可以确保每次组件被渲染时，钩子都会以相同的顺序被调用。这可以使得React正确保存多个useState和useEffect调用之间的Hooks状态。

Rule 2: Only call Hooks from React Functions

不要从常规的JavaScript函数中调用hook。相反：

  • 从React函数组件中调用钩子
  • 从自定义hook调用hook

Migrating a class component to React Hooks

让我们对比一下类组件和使用Hooks的函数组件：

// 依赖
import { Component } from 'react'
import axios from 'axios'
// TypeScript 定义 Types
type Issue = {
  number: number
  title: string
  state: string
}
// 类参数
type Props = {}
// 类状态
type State = { issues: Issue[] };
class Issues extends Component<Props, State> {
  // 类构造函数
  constructor(props: Props) {
    // 父类构造函数
    super(props)
    // 初始化类状态
    this.state = {
      issues: []
    }
  }
  // 生命周期函数
  componentDidMount() {
    axios
      .get('https://api.github.com/repos/ContentPI/ContentPI/issues')
      .then((response: any) => {
        this.setState({
          issues: response.data
        })
      })
  }
// 渲染函数
  render() {
    const { issues } = this.state
    return (
      <>
        <h1>ContentPI Issues</h1>
        {issues.map((issue: Issue) => (
          <p key={issue.title}>
            <strong>#{issue.number}</strong> {' '}
            <a href=
              {`https://github.com/ContentPI/ContentPI/issues/${issue.number}`}
              target="_blank">{issue.title}</a> {' '}
            {issue.state}
          </p>
        ))}
      </>
    )
  }
}
export default Issues

现在，让我们慢慢把他改为函数组件：

首先我们需要引入一些函数组件所需依赖：

// Dependencies
import { FC, useState, useEffect } from ' react'
import axios from ' axios'

因为函数组件没有props和state，因此在TypeScript中只留下Issue的定义即可：

// Types
type Issue = {
  number: number
  title: string
  state: string
}

然后，我们将类组件的定义改为函数组件：

const Issues: FC = () => {. . . }

FC类型用于使用 TypeScript 定义一个函数组件。如果你需要传递props给组件，你可以像这样传递它们:

type Props = {
  propX: string
  propY: number
  propZ: boolean
}
const Issues: FC<Props> = () => {. . . }

接下来要做的是使用useState钩子替换构造函数和状态定义:

// The useState hook replace the this. setState method
const [issues, setIssues] = useState<Issue[]>([])

其中 issues 为状态，setIssues为设置器，接受 Issue 数组，且初始化为空数组

我们之前使用componentDidMount生命周期方法，该方法在组件被挂载时执行，并且只运行一次。在 React Hook中，我们使用useEffect 通过不同的语法处理所有的生命周期方法。让我们看看如何使用 useEffect获得与componentDidMount相同的效果:

useEffect(() => {
  axios
    .get('https://api.github.com/repos/ContentPI/ContentPI/issues')
    .then((response: any) => {
      // Here we update directly our issue state
      setIssues(response.data)
    })
}, [])

最后，我们返回渲染的代码即可：

return (
  <>
  <h1>ContentPI Issues</h1>
  {issues.map((issue: Issue) => (
  <p key={issue.title}>
    <strong>#{issue.number}</strong> {' '}
    <a href=
      {`https://github.com/ContentPI/ContentPI/issues/${issue.number}`}
      target="_blank">{issue.title}</a> {' '}
    {issue.state}
  </p>
))}
  </>
)

至此，我们就彻底实现了从类组件到函数组件的改变，总览代码如下所示：

// 引入所需依赖
import { FC, useState, useEffect } from 'react'
import axios from 'axios'

// 使用 TypeScript 定义 Issue
type Issue = {
  number: number
  title: string
  state: string
}

// 创建函数组件
const Issues: FC = () => {
  // 使用 useState 创建状态和设置器
  const [issues, setIssues] = useState<Issue[]>([])

  // 使用 useEffect 管理生命周期函数
  useEffect(() => {
      axios
        .get('https://api.github.com/repos/ContentPI/ContentPI/issues')
        // 调用 状态设置器 setIssues 更新状态
        .then((response: any) => {
          setIssues(response.data)
        })
    }, []
  )

// 返回渲染结果
  return (
    <>
      <h1>ContentPI Issues</h1>
      {issues.map((issue: Issue) => (
        <p key={issue.title}>
          <strong>#{issue.number}</strong> {' '}
          <a 
            href={`https://github.com/ContentPI/ContentPI/issues/${issue.number}`}
            target="_blank">{issue.title}</a> {' '}
          {issue.state}
        </p>
      ))}
    </>
  )
}

export default Issues

Understanding React effects

在本节中，我们将学习我们在类组件上使用的组件生命周期方法与新的React effects 之间的区别。你可能在其他地方了解到它们是一样的，只是语法不同而已，然而这是不正确的。

Understanding useEffect

当你使用useEffect时，你需要think in effects。如果你想使用useEffect执行componentDidMount的等效方法，你可以这样做:

useEffect(() => {
// Here you perform your side effect
}, [])

第一个参数是想要执行的回调函数，第二个参数是依赖项数组。如果你给依赖项传递一个空数组([])，state和props使用初始值。如果你传递了一个依赖数组，useEffect钩子只会在其中一个依赖改变时运行。

useEffect与componentDidMount、componentDidUpdate不同的是，我们传递给useEffect的函数在布局(layout)和绘制(print)之后，在延迟事件期间触发。在一些需要同步触发的特殊形况下（如修改文档对象模型(DOM)，因为异步触发会出现闪烁），会产生期望之外的效果。

React提供了一个名为uselayouteeffect的钩子，它的工作方式与useEffect完全相同，但是可以与layout、print同步触发。

Understanding useCallback, useMemo, and memo

为了理解useCallback、usemmo和memo之间的区别，我们将做一个待办事项列表的例子。你可以使用create-react-app和typescript作为模板来创建一个基本的应用程序:

create-react-app todo --template typescript

在那之后，你可以删除src目录下所有额外的文件( App.css , App.test.ts , index.css , logo.svg , reportWebVitals.ts , setupTests.ts等等)，只需要保留App. tsx文件，并将其中的代码修改为：

// 所需依赖
import { useState, useEffect, useMemo, useCallback } from 'react'

// 所需组件
import List, { Todo } from './List'

// 初始化待办列表
const initialTodos = [
  { id:1, task:'Go shopping'},
  { id:2, task:'Pay the electricity bill'}
]

// todolist的整体组件
function App() {
  // 通过useState 设置 todeList 状态和其设置器，todoList为所有的待办
  const [todoList, setTodoList] = useState(initialTodos)
  // 通过useState 设置 task 状态和其设置器，task为需要新增的待办
  const [task, setTask] = useState('')
  // 使用 useEffect 在其方法实行是输出提示
  useEffect(() => {
    console.log('Rendering <App />')
  })
  // 新增待办的方法
  const handleCreate = () => {
    const newTodo = {
      id:Date.now(),
      task
    }
    // 在这里将原列表展开，与新待办一起作为更新值
    setTodoList([...todoList, newTodo])
    // 清空 task
    setTask('')
  }
  return (
    <>
      <input
        type="text"
        value={task}
        // 当输入框变化时，更新task的值
        onChange={(e) => setTask(e.target.value)}
      />
      // 点击按钮时，新增待办
      <button onClick={handleCreate}>Create</button>
      <List todoList={todoList} />
    </>
  )
}
export default App

在返回的渲染函数中，我们使用的List组件，现在我们就来同级目录下创建一个List.tsx文件：

// 引入所需依赖
import { FC, useEffect } from 'react'
// 引入所需组件
import Task from './Task'
// 定义单个待办的类型
export type Todo = {
  id: number
  task: string
}
// 定义函数属性的类型
interface Props {
  todoList: Todo[]
}
// 定义 List 函数组件，其中接受Props作为参数，这里的 {todoList}表示从Props中提取key为todoList的值（即Todo[]）
const List: FC<Props> = ({ todoList }) => {
  useEffect(() => {
    // 每一次新渲染都会输出提示
    console.log('Rendering <List />')
  })

  return (
    <ul>
      // 遍历 Todo[]，将其以下述形式展示
      {todoList.map((todo: Todo) => (
        <Task key={todo.id} id={todo.id} task={todo.task} />
      ))}
    </ul>
  )
}
export default List

在上述渲染中，我们使用了Task组件，同样在同级目录下创建一个Task.tsx的文件：

// 引入依赖
import { FC, useEffect } from 'react'

// 定义函数组件属性接口
interface Props {
  id: number
  task: string
}

// 创建函数组件
const Task: FC<Props> = ({ task }) => {
  useEffect(() => {
    // 每次渲染时都会调用
    console.log('Rendering <Task />', task)
  })
  return (
    <li>{task}</li>
  )
}
export default Task

ReactDOM.render(

启动项目，打开控制看，看看组件的渲染是如何进行的：

如图所示：当我们渲染待办事项列表时，默认情况下，我们正在执行Task组件的两次渲染，list的渲染一次，App组件的渲染一次。

当我们在输入框中输入文本，组件会如何更新渲染呢？

我们发现，当输入G, o时，我们又重复了两次上面的渲染方式，但实际上，在没有真正添加新待办时，List组件和已经存在的两个Task组件应该不用重新渲染，我们可以使用 memo 来改善这个不足。

Memoizing a component with memo

memo 是一种高阶组件，有点类似类组件中的纯组件（PureComponent），memo 会在首次渲染时维持prop信息，并在每次重新渲染前将prop进行浅比较（shallow comparison），只有在prop改变时，才会重新渲染组件。

我们只需要引入 memo，并在对存在上述问题的组件（List、Task）进行增强即可

可以看到，当仅改变输入框内容时，只有App被重新渲染了，这是合理的，因为输入框内容是App的一部分。

那为什么React 不将这种条件渲染设为默认呢，其实，浅比较的代价往往比重新渲染要高，因此作者给出了一个自己的建议: just don’t use it，除非每次渲染都需要重新请求API，或者重新渲染部分的数据量相当大（通常是体量相当大的列表项）

Memoizing a value with useMemo

现在，让我们为待办列表添加一个搜索功能，我们首先需要向App 中添加一个状态：term

const [term, setTerm] = useState(' ')

再添加一个用来处理搜索的函数handleSearch：

const handleSearch = () => {
  setTerm(task)
}

接着，我们新建一个变量filteredTodoList，用来来保存我们过滤后的列表项：

const filteredTodoList = todoList.filter((todo: Todo) => {
  console.log('Filtering...')
  return todo.task.toLowerCase().includes(term.toLowerCase())
})

最后，将我们展示的列表换成筛选后的列表：

<List todoList={filteredTodoList} />

现在，我们尝试在输入框里输入信息，你会发现：

每输入一个字符，Filter都会被调用两次（但我不明白为什么会调用两次），为了优化这一行为，可以使用 useMemo

const filteredTodoList = useMemo(
  () => todoList.filter((todo: Todo) => {
    console.log('Filtering...')
    return todo.task.toLowerCase().includes(term.toLowerCase())
  }),
  [term, todoList]
)

注意这里我们向第二个数组中传入了term, todoList，其中term是指当term发生变化时（即点击search按钮）或者todoList变化时（即点击create按钮），才触发回调函数的执行

现在，我们再次输入框中写入一些内容，可以看到filter不会像以前那样一直执行，同时添加和搜索都可以正确执行。

和memo类似。useMemo也是just don’t use it

Memoizing a function definition with useCallback

现在，为我们的待办列表添加一个删除的功能来学习useCallback。首先，添加一个删除函数：

const handleDelete = (taskId: number) => {
  const newTodoList = todoList.filter((todo: Todo) => todo.id !== taskId)
  setTodoList(newTodoList)
}

接着，我们需要从App中将该函数传递给List：

<List todoList={filteredTodoList} handleDelete={handleDelete} />

由于在List里使用TypeScript定义过Prop的类型，因此这里需要把我们新添加的类型补上：

interface Props {
  todoList: Todo[],
  handleDelete: (taskId: number) => void
}

由于List在使用Prop时使用的是解构赋值，因此这里需要将新添加的handleDelete解构出来：

const List: FC<Props> = ({ todoList, handleDelete }) => {...}

不仅如此，我们的List还需要将该函数再向下传递给Task：

{todoList.map((todo: Todo) => (
  <Task key={todo.id} id={todo.id} task={todo.task} handleDelete={handleDelete} />
))}

同样，由于Task里也通过TypeScript 定义了Prop的类型，因此也需要修改：

interface Props {
  id: number
  task: string
  handleDelete: (taskId:number) => void
}

到此，删除函数就从App传递给Task了，我们在Task渲染的单项中添加一个删除按钮即可：

<li>{task}<button onClick={() => handleDelete(id)}>x</button></li>

现在，删除功能就实现了，我们可以把Go shopping 给删除：

目前为止，似乎一切都很顺利，但是让我们在输入框中输入一些文字，观察控制台：

我们发现每次输入都会重新渲染所有组件，文中说是因为函数被传递到多个组件中，

（At this point, you are probably thinking about, what is going on if we have already implemented the memo HOC to memorize the components? But the problem now is that our handleDelete function is being passed in two components, from App to List and to Task , and the issue is that this function is regenerated every time we have a new re-render, in this case, every time we write something. So how do we fix this problem?）

在这种情况下，可以使用 useCallback 钩子，它在语法上与useMemo非常相似，但主要的区别是，它不是像useMemo那样记住函数的结果值，而是记住函数的定义:

我们的handleDelete应该改成这样：

const handleDelete = useCallback((taskId: number) => {
  const newTodoList = todoList.filter((todo: Todo) => todo.id! == taskId)
  setTodoList(newTodoList)
}, [todoList])
